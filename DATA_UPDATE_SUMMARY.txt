=== CHANGES MADE ===

1. **1 Year Real Data**
   - CFG["backtest_days"] = 365
   - System now requests up to 1 full year of historical data

2. **Supported Timeframes**
   - "1m", "3m", "5m", "15m"
   - Default = "5m" (you can change it)

3. **Real Data First (Priority)**
   - Always tries Yahoo Finance (yfinance) real candles first
   - Uses correct interval for each timeframe
   - Requests "max" history available
   - Only uses synthetic data if Yahoo returns nothing

4. **How it works now**
   - fetch(symbol, tf, days) → uses CFG if not specified
   - _fetch_yahoo → maps "1m"→"1m", "3m"→"3m" etc.
   - All backtests & signals use the configured TF

5. **Important Reality Check (Yahoo Limits)**
   - 5m & 15m → usually can get 60 days to 1+ year
   - 1m & 3m → Yahoo usually limits to last 7-30 days only
   - The app will automatically get whatever real data is available
   - Synthetic data is only fallback

=== HOW TO USE ===

To change default timeframe:
Edit app.py → CFG section:

CFG = {
    "backtest_days": 365,
    "timeframes": ["1m", "3m", "5m", "15m"],
    "default_tf": "15m",      # ← change this to 1m / 3m / 5m / 15m
}

Then run:
python app.py

The dashboard will now run on real data (up to 1 year where possible) on your chosen timeframe.
